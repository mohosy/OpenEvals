# Example Suites

These suites are meant to be copied, forked, and adapted.

## Included examples

- [`support-routing.yaml`](./support-routing.yaml): route support tickets into queues with deterministic and judge-based checks
- [`structured-extraction.yaml`](./structured-extraction.yaml): extract structured data into stable JSON
- [`summarization-clarity.yaml`](./summarization-clarity.yaml): compare concise summaries for clarity and completeness

## How to contribute a suite

Good community suites usually have:

- A clearly named problem domain
- At least 3-5 realistic cases
- Deterministic checks where possible
- One or two rubric criteria for nuance
- Public-safe sample inputs

If you create a reusable suite:

1. Post it in GitHub Discussions.
2. Open a benchmark-suite issue if you want it included here.
3. Add a short README note explaining what “good” looks like for that domain.
