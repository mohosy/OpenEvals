# Launch Playbook

If the goal is organic GitHub growth, the launch surface matters as much as the code.

## Before making the repo public

- Replace placeholder GitHub links in the app and README with the real repository URL.
- Add GitHub topics such as `llm`, `evals`, `openai`, `prompt-engineering`, `react`, and `fastapi`.
- Capture two screenshots:
  - The suite editor with YAML and run controls.
  - The compare screen with two variants and regression metrics.
- Record a 30-45 second silent demo GIF that shows import, run, compare, and baseline pinning.

## Launch artifacts

- Pin a short GitHub Discussion that asks for shared public eval suites.
- Publish at least three benchmark repos or gists using the example YAML format.
- Post the compare screenshot and a one-sentence problem statement in the README hero.
- Ship a first `v0.1.0` release on GitHub with release notes and the demo GIF.

## Launch message

Use a sharp framing:

> OpenEvals is a beautiful open-source UI for prompt evals. Import YAML, compare `gpt-4o` vs `gpt-4.1`, pin a baseline, and run the same suite in GitHub Actions.
