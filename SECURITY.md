# Security Policy

## Supported Versions

OpenEvals is currently maintained as a moving `main` branch plus tagged releases.

## Reporting a Vulnerability

Do not open public GitHub issues for security-sensitive reports.

Instead:

1. Open a private GitHub security advisory if available.
2. If that is not available, contact the repository maintainers privately through the contact channel listed in the repository profile.

Include:

- A clear description of the issue
- Reproduction steps or a proof of concept
- Impact assessment
- Any suggested mitigations

We will acknowledge valid reports and work on a fix before public disclosure.
